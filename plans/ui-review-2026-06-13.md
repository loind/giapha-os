## 🔴 Critical Issues

### C-1: Dashboard primary buttons màu đen không nhất quán với amber CTA
**Severity**: High
**Finding**: Nhiều primary action buttons trên dashboard dùng `bg-stone-900` (đen) thay vì amber:
- Events page: "Thêm sự kiện" button → đen
- Gallery page: "Thêm hình ảnh" button → đen
- Users page: "Thêm người dùng" button → đen
- Lineage page: "Tính toán" button → đen
- Data page: "Xuất JSON" button → đen
- Login page: "Đăng nhập" button → đen

Trong khi landing page CTA đã dùng amber-600. Tạo sự không nhất quán về brand identity.

**Fix**: Đổi tất cả primary action buttons từ `bg-stone-900` → `bg-amber-600 hover:bg-amber-700 text-white`. Áp dụng `.btn-cta` class đã định nghĩa trong `globals.css`.

## 🟡 High Priority

### H-1: Login page thiếu ThemeToggle
**Severity**: Medium-High
**File**: `app/login/page.tsx`
**Finding**: Landing page có ThemeToggle trong header, nhưng login page chỉ có "Trang chủ" và "Giới thiệu" buttons. User không thể đổi theme khi đang ở login page.
**Fix**: Thay nút "Giới thiệu" ở top-right bằng ThemeToggle, hoặc thêm ThemeToggle cạnh nó.

### H-2: Login page — Password field thiếu toggle show/hide
**Severity**: Medium
**File**: `app/login/page.tsx`
**Finding**: Không có nút show/hide password. Đây là standard UX pattern cho login forms.
**Fix**: Thêm nút Eye/EyeOff icon bên phải password input để toggle visibility.

### H-3: About page thiếu ThemeToggle và back to landing
**Severity**: Medium
**File**: `app/about/page.tsx`
**Finding**: About page chỉ có "Quay lại" button (về dashboard — chỉ dành cho authenticated users). Non-authenticated users vào /about không có cách về landing page.
**Fix**: Đổi "Quay lại" → link về "/" thay vì "/dashboard". Thêm ThemeToggle.

### H-4: Setup page — Code block quá tối, khó đọc
**Severity**: Medium
**File**: `app/setup/page.tsx`
**Finding**: SQL code block có background quá tối (gần đen), contrast thấp với text. Khó đọc trên mobile.
**Fix**: Dùng màu nền nhẹ hơn (stone-800 thay vì gần đen), tăng font size, thêm syntax highlighting.
